HOW TO READ THE RESULTS — GUIDE FOR PRESENTATION
📋 Classification Report

Shows per-class breakdown for each of the 7 categories. Key columns to quote in your viva:

precision = TP / (TP + FP)
recall = TP / (TP + FN)
f1-score = 2 × (P × R) / (P + R)
📊 Confusion Matrix

A 7×7 grid where rows = actual labels, columns = predicted labels. Strong diagonal means high accuracy. Off-diagonal numbers are misclassifications.

accuracy = correct predictions / total samples
🎯 Precision vs Recall

High Precision — when model says "Positive", it is usually right.
High Recall — the model finds most of the actual Positive comments.
F1-Score balances both — use this as your accuracy number in presentation.

✅ Macro vs Weighted Average

Macro avg — simple mean across all 7 classes. Good when classes are balanced (ours are: 7143 per class).
Weighted avg — weighted by class size. Both values should be similar for our balanced dataset.

HYPERPARAMETERS — WHAT THEY MEAN & WHY WE CHOSE THESE VALUES
ParameterValue UsedWhat it does & why we chose it
n_estimators80 Number of decision trees in the forest. More trees = more stable predictions. 80 gives good accuracy without excessive training time (1–3 min).
max_features (TF-IDF)1500 Maximum number of unique words kept by the vectorizer. Limiting to 1500 reduces memory and overfitting while keeping the most informative tokens.
stop_wordsEnglish (NLTK) Common words like "the", "is", "and" are removed before vectorizing since they appear in all categories and carry no sentiment information.
test_size0.20 20% of 50K = 10,000 samples are held out for testing. The model never sees these during training — ensures unbiased evaluation.
random_state0 Fixes the random seed so results are fully reproducible. Running the training script twice gives the exact same model and accuracy.
⚡ LIKELY VIVA QUESTIONS — CLICK TO EXPAND ANSWERS
Why did you choose Random Forest over other algorithms? ▶
Random Forest is an ensemble method combining 80 decision trees. Individual trees overfit to noise, but when we aggregate their votes, noise cancels out and the model generalises well. For our 7-class text classification problem, Random Forest gave the best trade-off between training speed and accuracy. It also handles the high-dimensional TF-IDF feature space (1500 features) well without requiring feature scaling like SVM does.
What is TF-IDF and why not use Word2Vec or BERT? ▶
TF-IDF (Term Frequency–Inverse Document Frequency) converts text to numerical vectors by weighting words based on how unique they are to a document. It is fast, interpretable, and works well with tree-based models. Word2Vec and BERT are more powerful but require significantly more compute and training data. For our synthetic 50K dataset with template-based comments, TF-IDF is sufficient and appropriate. BERT is listed as a Future Scope improvement in our report.
Why did you use a synthetic dataset instead of real YouTube comments? ▶
Using real YouTube comments requires a YouTube Data API key subject to quota limits of 10,000 units per day. Manually labelling real comments across 7 categories is extremely time-consuming and subjective. Our synthetic dataset generated by generate_dataset.py provides perfectly balanced labels at 7,143 per class, is free from API limitations, and gives full control over category definitions. The trade-off is reduced vocabulary diversity — acknowledged in our report's Future Scope.
What does the confusion matrix diagonal tell you? ▶
The main diagonal shows correct predictions — where the actual class equals the predicted class. A strong bright diagonal means the model is correctly classifying each of the 7 categories. Off-diagonal cells are misclassifications — for example if cell row 0, column 2 is bright, it means Positive comments are sometimes predicted as Neutral. We want the diagonal as bright as possible and all other cells as dark as possible.
How does your app fetch comments without a YouTube API key? ▶
We use the open-source python library youtube-comment-downloader which scrapes YouTube's public comment section directly without requiring a Google API key. In classifier.py we call get_comments_from_url() sorted by SORT_BY_POPULAR and fetch up to 1,500 top comments per video using Python's islice. This avoids API quotas entirely and works on any public YouTube video URL.
What is the Singleton design pattern used in classifier.py? ▶
A Singleton ensures only one instance of a class exists in memory at a time. In our app, ClassiferSingleton uses a thread-safe lock so the Random Forest model and TF-IDF vectorizer are loaded from disk only once when the Flask server starts. Every request after that reuses the same in-memory object — saving 2 to 3 seconds per request that would otherwise be spent reloading the model files.
