ABSTRACT

The extensive user engagement on YouTube leads to a flood of comments, creating challenges for content creators who aim to understand audience sentiment. Previous studies have mainly concentrated on distinguishing between positive, negative, and neutral sentiments in comments. This study involves creating a dataset of YouTube comments and classifying them into positive, negative, interrogative, corrective, imperative, and neutral categories, which will help content creators visualize other sentiments alongside positive, negative, and neutral sentiments.

In this project, we present a web application developed using the Flask framework, designed to classify YouTube comments for a specific video URL into multiple categories using machine learning. The application aims to provide content creators, marketers, and researchers with a powerful tool for analyzing viewer feedback, understanding audience sentiment, and identifying trends.

ML Technology Insights
🌲

RANDOM FOREST CLASSIFIER

An ensemble of 80 independent decision trees. Each tree votes on the class — the majority vote wins. This reduces overfitting compared to a single decision tree and achieves higher accuracy on unseen data.

ALGORITHM USED
📐

TF-IDF VECTORIZATION

Term Frequency–Inverse Document Frequency converts raw comment text into 1500 numerical features. Words common across all documents (like "the") get lower weight; unique discriminative words get higher weight.

FEATURE EXTRACTION
🧹

NLP PREPROCESSING PIPELINE

Raw comments go through: (1) Remove special characters using regex, (2) Remove single-letter tokens, (3) Collapse whitespace, (4) Lowercase all text, (5) Remove NLTK English stopwords before vectorizing.

TEXT CLEANING
🎲

BOOTSTRAPPING (BAGGING)

Each of the 80 trees is trained on a different random sample of the 50K dataset (with replacement). This technique — called Bootstrap Aggregating or Bagging — ensures diversity among trees and improves model robustness.

ENSEMBLE TECHNIQUE
🔗

SINGLETON PATTERN IN CODE

The classifier uses a thread-safe Singleton design pattern so the model is loaded into memory only once at startup. Subsequent requests reuse the same in-memory object — saving 2–3 seconds per request.

SOFTWARE DESIGN
📦

JOBLIB MODEL SERIALIZATION

The trained Random Forest model and TF-IDF vectorizer are saved as .sav files using joblib. This allows the app to skip retraining on every startup — the pre-trained model loads in under 1 second.
